# Retrospective: 2026-05-30 — !dlchart command

## Commit covered
- `9381c8b` Add !dlchart command to download latest Beatport monthly chart

---

## What went well

- Two-stage fetch (charts listing → chart page) cleanly solves the
  dynamic URL problem without any hardcoded IDs.
- Refactored `fetchNextData`, `parseQueries`, and `normaliseTracks` into
  shared helpers — `fetchNewReleases` and `fetchChartTracks` both benefit
  and beatport.js is less repetitive.
- [Download All] + numbered individual picks gives full flexibility
  without needing a separate command.
- Genre selection deferred until after chart pick, so the user sees what
  they're getting before committing to a folder.

---

## What could be better

### 1. Still fragile against Beatport page structure changes
Both chart discovery and track extraction rely on `__NEXT_DATA__` parsing.
One Beatport deploy can break both. No automated check to detect breakage.

### 2. Chart slug pattern is hardcoded
`'best-new-hard-techno'` is hardcoded in `fetchLatestChart`. If Beatport
renames the chart series, the function returns nothing. Should be
configurable via .env (e.g. BEATPORT_CHART_SLUG=best-new-hard-techno).

### 3. Button row count not guarded against >5 rows
A chart with more than 20 tracks would produce >5 rows and hit Discord's
limit. The `rows.slice(0, 5)` guards against the crash but silently
drops tracks beyond #20. Should inform the user tracks were truncated.

### 4. No deduplication against existing library
!dlchart downloads everything on the chart without checking whether
those files already exist in the output folder. Could skip tracks
already present (by matching filename or AcoustID).

---

## Rules to carry forward

1. **Externalise slug patterns and scraping selectors into config.**
   Anything that targets a specific third-party page structure will break —
   make it easy to fix without a code change.
2. **Always guard Discord component row counts** and tell the user when
   content is truncated rather than silently dropping it.
3. **Check for existing files before re-downloading** in any bulk
   download operation.
