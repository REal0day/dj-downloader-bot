# Retrospective: 2026-05-30 — !dlreport command

## Commit covered
- `5348d8f` Add !dlreport command to list low-confidence and unscanned files

---

## What went well

- The ACOUSTID_SCORE tag stored during scan/download made this essentially free
  to implement — no re-reading audio, just reading ID3 tags (fast).
- Sorted worst-first (lowest score at top) so the most suspect files are
  immediately visible.
- 2000-char Discord message cap handled with truncation rather than crashing.
- Separating "low confidence" from "never scanned" gives the user two clear
  action paths: re-download vs. !dlscan.

---

## What could be better

### 1. Truncation loses data silently
If there are more files than fit in 2000 chars, the list is cut off with
"…(truncated)". The user has no way to see the rest without running !dlscan
again or checking the files manually. Should paginate or DM the full list.

### 2. No threshold configuration
80% is hardcoded in three places now (writeTags, handleScan counts, handleReport).
Should be a single constant or .env variable.

### 3. Report doesn't show artist/title from tags
The report shows filename + score but not the ID3 artist/title already embedded
in the file. Showing those would make it easier to spot mismatches without
opening the file.

---

## Rules to carry forward

1. **When you store state in a file, make it queryable later.** The ACOUSTID_SCORE
   tag enabled !dlreport for free. Design storage with future reads in mind.
2. **Deduplicate magic numbers.** The 80% threshold appears in multiple places —
   define it once (config or constant) and reference it everywhere.
3. **Truncation should tell the user how much was cut.** "…(truncated, 47 more)"
   beats "…(truncated)" — the user knows whether to care.
